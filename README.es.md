# Sitio web de QuenchWorks

[English](README.md) · [العربية](README.ar.md) · **Español**

El sitio de marketing y el catalogo en vivo de [QuenchWorks](https://github.com/quenchworks), un catalogo gratuito y endurecido 0-CVE de imagenes de contenedor y charts de Helm firmados. Es la puerta de entrada en [quench-works.com](https://quench-works.com), y el indice navegable de cada imagen y chart que publicamos.

Construido con **Astro 5 · Tailwind CSS v4 · TypeScript**, gestionado con **pnpm**, sobre Node **26** (consulta `.nvmrc`). Monocromo por diseno (tinta/papel), Inter + JetBrains Mono. El tema se aplica mediante design tokens, no con hex codificado.

## Desarrollo

```bash
pnpm install
pnpm dev        # http://localhost:4321  (runs the catalog sync first)
pnpm check      # astro + TypeScript checks
pnpm build      # static output to dist/  (runs the catalog sync first)
pnpm preview    # preview the production build
```

`pnpm dev` y `pnpm build` ejecutan ambos la sincronizacion del catalogo de forma automatica (`predev` / `prebuild`), de modo que las paginas `/images` y `/charts` siempre reflejan el catalogo real.

## Como se generan los datos del catalogo

`scripts/sync-catalog.mjs` es el puente entre los repositorios del catalogo y el sitio. Lee la source of truth de [quenchworks/images](https://github.com/quenchworks/images) (`catalog.yaml`) y [quenchworks/charts](https://github.com/quenchworks/charts), luego escribe `src/data/images.json` y `src/data/charts.json`. Las paginas del catalogo se renderizan a partir de esos archivos, asi que los conteos, las versiones, los digests y los indicadores de licencia nunca se mantienen a mano.

```bash
pnpm sync       # node scripts/sync-catalog.mjs
```

## Estructura

```
src/
  data/                 images.json + charts.json (generated by sync-catalog.mjs)
  pages/
    index.astro         the hero / landing page
    images/             the hardened-image catalog
    charts/             the signed-chart catalog
    docs/               how-to-verify and reference docs
    roadmap.astro       what's shipped and what's next
    security.astro      the security model and verification steps
    about.astro · how-it-works.astro
  layouts/ · components/ · styles/
scripts/sync-catalog.mjs   pulls catalog data from the images + charts repos
public/                    brand assets, favicon
```

## Tema

El color y la tipografia provienen de los design tokens en `src/styles/global.css` (`@theme`). No codifiques hex en los componentes; usa las utilidades `ink` / `graphite` / `line` / `ash` / `paper` para que todo el sitio se mantenga consistente. Solo monocromo, sin acento de color.

## Despliegue

Construccion estatica (`pnpm build` → `dist/`), desplegada en **Cloudflare Pages** mediante la integracion con Git (Cloudflare construye en cada push, sin necesidad de un workflow de GitHub Actions):

- Comando de construccion: `pnpm build`
- Directorio de salida de la construccion: `dist`
- Preset de framework: Astro
- Version de Node: `26` (configura `NODE_VERSION=26` en el entorno de Pages, coincidiendo con `.nvmrc`)
- Dominio personalizado: `quench-works.com`
